# SAP Mall CI/CD 自动部署方案

## 环境信息

- **服务器系统**: CentOS/RHEL
- **HTTPS**: 暂不配置
- **部署触发**: 手动触发 (workflow_dispatch)
- **数据库迁移**: 不需要

## 现状分析

项目已有基础 GitHub Actions 工作流 (`.github/workflows/deploy.yml`)，但存在以下可优化点：

| 问题 | 说明 |
|------|------|
| 无测试环节 | 没有 `npm test` / `go test` 步骤 |
| 无代码检查 | 没有 ESLint / golangci-lint |
| 无缓存 | 每次构建都重新下载依赖 |
| 无回滚机制 | 部署失败无法快速恢复 |
| 无版本管理 | 无法追踪部署历史 |

## 目标架构

```
GitHub 仓库
    │
    ├── 手动触发 ──→ GitHub Actions CI/CD
    │                    │
    │                    ├── 1. 构建产物 (Build)
    │                    ├── 2. 上传产物 (Artifacts)
    │                    └── 3. 部署到服务器 (Deploy)
    │
    └── CentOS 服务器
         ├── /opt/sapmall/
         │   ├── backend/        (Go 二进制)
         │   ├── frontend/
         │   │   ├── admin/      (React 构建)
         │   │   ├── dapp/       (React 构建)
         │   │   └── website/    (React 构建)
         │   ├── deploy/         (版本归档)
         │   └── backup/         (回滚备份)
         └── systemd services
             ├── sapmall-backend
             └── nginx
```

## 实现步骤

### 第一步：创建部署工作流文件

**文件**: `.github/workflows/deploy-prod.yml`

```yaml
name: Deploy Production

on:
  workflow_dispatch:  # 仅手动触发

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # 缓存依赖
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - uses: actions/setup-go@v5
        with:
          go-version: '1.23'
          cache: true

      # 构建前端
      - name: Build Admin
        run: cd web_client/sapmall-admin && npm ci && npm run build
      - name: Build DApp
        run: cd web_client/sapmall-dapp && npm ci && npm run build
      - name: Build Website
        run: cd web_client/sapmall-website && npm ci && npm run build

      # 构建后端
      - name: Build Backend
        run: |
          cd backend_service
          go mod tidy
          CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -installsuffix cgo -o main ./app/sapmall_start.go

      # 上传构建产物
      - name: Upload Artifacts
        uses: actions/upload-artifact@v4
        with:
          name: build-${{ github.sha }}
          path: |
            web_client/sapmall-admin/build/
            web_client/sapmall-dapp/build/
            web_client/sapmall-website/build/
            backend_service/main
          retention-days: 7

  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Download Artifacts
        uses: actions/download-artifact@v4
        with:
          name: build-${{ github.sha }}

      - name: Deploy to Server
        uses: easingthemes/ssh-deploy@v5
        with:
          SSH_PRIVATE_KEY: ${{ secrets.SERVER_SSH_KEY }}
          REMOTE_HOST: ${{ secrets.SERVER_HOST }}
          REMOTE_USER: ${{ secrets.SERVER_USER }}
          REMOTE_PORT: ${{ secrets.SERVER_PORT || '22' }}
          SOURCE: "sapmall-admin/build/,sapmall-dapp/build/,sapmall-website/build/,main"
          TARGET: "/opt/sapmall/deploy/${{ github.sha }}/"
          ARGS: "-avz"

      - name: Activate Deployment
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          password: ${{ secrets.SERVER_PASSWORD }}
          port: ${{ secrets.SERVER_PORT || '22' }}
          script: |
            # 创建版本目录
            mkdir -p /opt/sapmall/backup
            
            # 备份当前版本
            if [ -d /opt/sapmall/frontend ]; then
              cp -r /opt/sapmall/frontend /opt/sapmall/backup/frontend_$(date +%Y%m%d_%H%M%S)
            fi
            
            # 部署新版本
            mkdir -p /opt/sapmall/frontend/{admin,dapp,website}
            cp -rf /opt/sapmall/deploy/${{ github.sha }}/sapmall-admin/build/* /opt/sapmall/frontend/admin/
            cp -rf /opt/sapmall/deploy/${{ github.sha }}/sapmall-dapp/build/* /opt/sapmall/frontend/dapp/
            cp -rf /opt/sapmall/deploy/${{ github.sha }}/sapmall-website/build/* /opt/sapmall/frontend/website/
            
            # 部署后端
            chmod +x /opt/sapmall/deploy/${{ github.sha }}/main
            cp /opt/sapmall/deploy/${{ github.sha }}/main /opt/sapmall/backend/main
            
            # 重启服务
            sudo systemctl restart sapmall-backend
            sudo systemctl reload nginx
            
            # 健康检查
            sleep 3
            curl -f http://localhost:8888/api/health || {
              echo "Health check failed, rolling back..."
              LATEST_BACKUP=$(ls -td /opt/sapmall/backup/frontend_* | head -1)
              if [ -n "$LATEST_BACKUP" ]; then
                cp -rf "$LATEST_BACKUP"/* /opt/sapmall/frontend/
                sudo systemctl restart sapmall-backend
              fi
              exit 1
            }
            
            echo "Deployment successful: ${{ github.sha }}"
```

### 第二步：服务器 Systemd 服务配置

在 CentOS 服务器上创建 systemd 服务文件：

```bash
# /etc/systemd/system/sapmall-backend.service
[Unit]
Description=SAP Mall Backend Service
After=network.target mysql.service redis.service

[Service]
Type=simple
User=sapmall
Group=sapmall
WorkingDirectory=/opt/sapmall/backend
ExecStart=/opt/sapmall/backend/main -f /opt/sapmall/config/sapmall_prod.yaml
Restart=always
RestartSec=5
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```

启用服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable sapmall-backend
sudo systemctl start sapmall-backend
```

### 第三步：Nginx 配置

更新 `nginx.conf.production`，确保路径与部署路径一致：

```nginx
# 静态资源缓存
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

### 第四步：GitHub Secrets 配置

在 GitHub 仓库 Settings → Secrets and variables → Actions 中添加：

| Secret 名称 | 说明 |
|-------------|------|
| `SERVER_SSH_KEY` | 服务器 SSH 私钥 |
| `SERVER_HOST` | 服务器 IP 地址 |
| `SERVER_USER` | SSH 用户名 |
| `SERVER_PORT` | SSH 端口 (默认 22) |
| `SERVER_PASSWORD` | SSH 密码 (备用) |

### 第五步：服务器目录初始化

```bash
# 创建目录结构
sudo mkdir -p /opt/sapmall/{backend,frontend,deploy,backup,config}
sudo mkdir -p /opt/sapmall/frontend/{admin,dapp,website}

# 创建 sapmall 用户
sudo useradd -r -s /bin/false sapmall
sudo chown -R sapmall:sapmall /opt/sapmall
```

## 验证步骤

1. **推送代码到 GitHub**
   ```bash
   git add .github/workflows/deploy-prod.yml
   git commit -m "ci: add production deployment workflow"
   git push origin main
   ```

2. **配置 GitHub Secrets**
   - 进入仓库 Settings → Secrets and variables → Actions
   - 添加 `SERVER_SSH_KEY`, `SERVER_HOST`, `SERVER_USER`, `SERVER_PASSWORD`

3. **手动触发部署**
   - 进入 Actions → Deploy Production
   - 点击 "Run workflow"
   - 选择 main 分支，点击 "Run workflow"

4. **服务器验证**
   ```bash
   # 检查服务状态
   sudo systemctl status sapmall-backend
   sudo systemctl status nginx
   
   # 检查健康
   curl http://localhost:8888/api/health
   curl http://localhost:7101/
   ```
