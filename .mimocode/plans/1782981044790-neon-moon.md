2. 删除 `sapmall-admin/public/index.html` 中的 Font Awesome CDN link
3. 从 `package.json` 中移除 `@fortawesome/*` 和 `@ant-design/icons` 依赖

## Phase 2: 品牌色重新定义 [P1]

### 2.1 目标
去除 AI 紫光渐变 (`#3b82f6 → #8b5cf6`)，建立有辨识度的品牌色。

### 2.2 已确定方案：C. 纯钴蓝

- **主色**：`#3b82f6` (Blue 500) 保留
- **改动**：去掉 `#8b5cf6` (Violet 500)，所有蓝紫渐变改为纯蓝或蓝到青 (`#3b82f6 → #06b6d4`)
- **原则**：单一强调色体系，不再混入紫色

### 2.3 实施内容
1. 更新三个 app 的 `tailwind.config.js` — 保留 `sapphire` 色板，移除紫色引用
2. 替换所有 `linear-gradient(135deg, #3b82f6, #8b5cf6)` 为纯蓝或蓝到青渐变
3. 统一按钮、链接、高亮的强调色为 `#3b82f6`
4. 更新 `dapp-common.css`、`admin-common.scss` 中的紫色引用
5. 更新 DAO (`--dao-primary: #8b5cf6` → `#3b82f6`)、Help (`--help-primary: #8b5cf6` → `#3b82f6`) 模块的 CSS 自定义属性
6. Admin 的 `chainnetTheme` 中 `colorPrimary: #34d399` 保持不变（已是翡翠色，与蓝不冲突）

## Phase 3: 字体系统升级 [P1]

### 3.1 目标
自托管字体，统一字体策略。

### 3.2 实施内容
1. 将 Google Fonts 改为 `@font-face` + `font-display: swap` 自托管
2. 移除 `index.html` 中的 Google Fonts `<link>` 标签
3. 移除 SCSS 中的 `@import url()` 字体加载
4. 评估 `Cormorant Garamond` 的使用——如果保留则正式纳入字体系统，否则用 `Plus Jakarta Sans` italic 替代
5. 统一 `Exo 2` 和 `Plus Jakarta Sans` 的使用场景

### 3.3 关键文件
- `sapmall-dapp/public/index.html`
- `sapmall-dapp/src/pages/dao/DaoLayout.module.scss`
- `sapmall-dapp/public/ecosystem_activity.html`
- `sapmall-website/public/index.html`（如有）

## Phase 4: 动效系统引入 [P1]

### 4.1 目标
引入 Motion（Framer Motion 新版本），添加关键交互动效。

### 4.2 实施内容
1. 安装 `motion` 包（`motion/react`）
2. 为 Hero section 添加入场动画（stagger reveal）
3. 为卡片添加 `whileInView` 滚动入场
4. 为按钮添加 `whileHover` / `whileTap` 物理反馈
5. 为页面切换添加过渡效果
6. 所有动效尊重 `prefers-reduced-motion`

### 4.3 优先页面
1. dapp — Marketplace 首页（产品卡片入场）
2. dapp — DAO Hero（已有 CSS 动画，升级为 Motion）
3. dapp — Rewards 页面
4. website — HomePage（全页滚动入场）

## Phase 5: 布局多样化 [P2]

### 5.1 目标
打破模板化布局，增加构图变化。

### 5.2 实施内容
1. Website `HomePage.tsx` — 将对称三列 feature cards 改为 bento grid

## Phase 6: Glassmorphism 层次化 [P2]

### 6.1 目标
建立视觉纵深，避免全站统一模糊度。

### 6.2 实施内容
1. 顶层容器（header、modal、sidebar）— 保留强 blur（16-25px）
2. 次级卡片 — 降低 blur（4-8px）或改用实色背景 + 微妙边框
3. 添加 `@media (prefers-reduced-transparency: reduce)` 回退
4. 统一 `border-radius` 体系（当前 4px/6px/8px/12px/16px/999px 混用）

## 执行顺序

```
Phase 1 (图标统一) → Phase 2 (品牌色) → Phase 3 (字体) → Phase 4 (动效) → Phase 5-8 (并行)
     P0                 P1                 P1              P1              P2/P3
```

Phase 1 是独立的，可以立即开始。Phase 2-3 有依赖关系（颜色和字体变更影响全局样式）。Phase 4 依赖 Phase 1 完成（图标统一后引入动效更安全）。Phase 5-8 可以在 Phase 2-4 完成后并行推进。
