# 优化方案：卖家地址从商品 Creator 动态获取

## 背景

当前 `sellerAddress`（卖家收款地址）从配置文件 `OrderPayment.PlatformSellerAddress` 读取，是一个固定值。但实际业务中，不同商品属于不同商家，卖家地址应该从商品的创建者（`ProductSpu.UserCode`，即商家钱包地址）动态获取。

## 数据模型分析

| 模型 | 关键字段 | 说明 |
|------|---------|------|
| `ProductSpu` | `UserID`, `UserCode` | `UserCode` 是商家钱包地址 |
| `ProductSku` | `ProductSpuID`, `ProductSpuCode` | 关联到 SPU |
| `OrderPayment` | 无 `seller_address` 字段 | **需要新增** |
| `Order` | `SpuId`, `SpuCode` | 已保存商品快照 |

## 修改方案

### 后端改动（5 个文件）

#### 1. `model/order_payment.go` — 新增 SellerAddress 字段

```go
type OrderPayment struct {
    // ... 现有字段 ...
    SellerAddress        string    `json:"sellerAddress" gorm:"column:seller_address"`
}
```

#### 2. `logic/order/create_order_logic.go` — 从 SPU 获取卖家地址

- 删除：`resolvePlatformSellerAddress(l.svcCtx.Config.OrderPayment.PlatformSellerAddress)`
- 新增：通过 `sku.ProductSpuID` 查询 `ProductSpu`，取 `UserCode` 作为卖家地址
- 将 `sellerAddress` 存入 `OrderPayment` 记录

改动位置：
- `validateCreateOrderAfterAuth()` 第 168 行：替换 sellerAddress 获取逻辑
- `assembleCreateOrderBundle()` 第 270 行：将 sellerAddress 写入 paymentRow

#### 3. `logic/order/get_order_logic.go` — 从 OrderPayment 读取

- 删除：`platformSellerAddress()` 方法（从配置读取）
- 改为：从 `paymentRow.SellerAddress` 读取

改动位置：
- `GetOrder()` 第 104 行：`toOrderPaymentInfo(&paymentRow, paymentRow.SellerAddress)`
- 删除第 113-119 行 `platformSellerAddress()` 方法

#### 4. `logic/order/order_helpers.go` — 删除废弃函数

- 删除：`resolvePlatformSellerAddress()` 函数（第 75-81 行）

#### 5. `config/config.go` — 可选：移除 OrderPayment.PlatformSellerAddress

- 保留配置项但不再使用（向后兼容），或直接移除

> 注意：`order_status_logic.go`、`modify_order_logic.go`、`async_confirm.go`、`list_order_logic.go`、`admin` 模块均未引用 `PlatformSellerAddress`，无需修改。

### 前端改动（0 个文件）

前端无需改动。前端已经从后端响应中读取 `sellerAddress` 字段：
- `orderApi.ts:93` — `OrderPaymentInfo.sellerAddress`
- `orderApi.ts:230` — `paymentToIntentBundle()` 映射
- `executePayOrder.ts:49` — 校验并传入合约

### 数据库变更

需要给 `sys_order_payment` 表新增 `seller_address` 字段：

```sql
ALTER TABLE sys_order_payment ADD COLUMN seller_address VARCHAR(128) DEFAULT '' AFTER payer_address;
```

1. 后端编译通过：`go build ./app/sapmall_start.go`
2. 创建订单时检查 `sys_order_payment.seller_address` 是否正确写入商品 creator 的钱包地址
3. 查询订单时 `sellerAddress` 字段返回正确值
4. 前端支付流程正常（payOrder 合约调用参数正确）
