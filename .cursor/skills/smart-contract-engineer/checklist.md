# Checklist - Smart Contract Engineer

## A. 需求确认
- [ ] 业务流程已抽象为状态机（创建/变更/结束）
- [ ] 角色与权限边界明确（谁可执行何操作）
- [ ] 链上与链下职责边界明确（哪些数据上链、哪些仅索引）

## B. 合约设计
- [ ] 合约拆分合理（核心逻辑/接口/库）
- [ ] 关键状态变化定义了事件
- [ ] 自定义错误（custom errors）用于高频 revert 场景
- [ ] 存储结构与可扩展性已考虑（升级或迁移策略）

## C. 安全与经济模型
- [ ] 外部调用顺序安全（避免重入）
- [ ] 敏感函数具备严格权限校验
- [ ] 参数上下限、边界值和异常路径已覆盖
- [ ] 关键经济参数可治理或可紧急暂停

## D. 测试覆盖
- [ ] 正常路径测试
- [ ] revert/异常测试
- [ ] 权限测试（越权拒绝）
- [ ] 边界值测试（0、最大值、重复调用）
- [ ] 事件断言（关键事件是否正确触发）

## E. 部署与配置
- [ ] 网络配置使用环境变量，不硬编码密钥
- [ ] 部署脚本参数可配置（地址、初值、管理员）
- [ ] 已记录部署产物（地址、版本、构造参数）

## F. 集成交付
- [ ] 提供 ABI/合约地址清单
- [ ] 提供事件字段解释与索引建议
- [ ] 提供前后端调用约定（单位、精度、状态码）
- [ ] 提供已知风险和后续优化项
